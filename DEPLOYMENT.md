# Deployment

Cloudflare Pages serves the site at https://rto.oriz.in (custom domain) and a default `*.pages.dev` subdomain.

## One-time setup

The CF Pages project (`rto-api` under the `oriz-org` Cloudflare account) is wired to GitHub via the Cloudflare GitHub App. Every push to `main` triggers a build.

After this repo gained an Astro build step, the Pages project must use these settings (Workers & Pages > rto-api > Settings > Builds):

| Setting | Value |
|---|---|
| Production branch | `main` |
| Build command | `npm run build` |
| Build output directory | `dist` |
| Root directory | _(empty)_ |
| Node version | `20` or later (set `NODE_VERSION=20` in Environment variables if needed) |

Before this change the project had no build command and served the raw repo root. **Updating the build command is required** &mdash; without it CF Pages will publish the source tree (including `src/`, `package.json`) instead of `dist/`.

## Local verification

```bash
npm install
npm run build
npx serve dist        # or:  python -m http.server -d dist 8080
```

Then curl a few endpoints:

```bash
curl http://localhost:8080/codes/MH-12.json
curl http://localhost:8080/states.json
curl -I http://localhost:8080/                 # should be HTML
curl -I http://localhost:8080/docs/            # should be HTML
curl -I http://localhost:8080/explorer/        # should be HTML
```

## Data regeneration

The JSON dataset is generated by `scripts/generate.cjs` from `scripts/build.cjs` and `scripts/data2.cjs`. To regenerate:

```bash
npm run data
```

Output goes into `public/`, which Astro copies verbatim into `dist/` on `npm run build`. Commit both the script changes and the regenerated `public/codes/`, `public/states/`, and `public/*.json` files.

## Custom domain

`rto.oriz.in` is a CNAME to the `*.pages.dev` subdomain. The TLS cert is issued by Cloudflare automatically; provisioning takes up to 15 min after first DNS resolution.
